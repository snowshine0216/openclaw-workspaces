## Answers

### 1. `delivery.mode`: `announce` vs `none`

| Mode | Behavior |
|------|----------|
| **`announce`** | - Delivers the isolated run's output directly to the target channel<br>- Posts a brief summary to the main session<br>- Suppresses messaging tool sends for the run (use `delivery.channel`/`delivery.to` instead) |
| **`none`** | - Internal only - **no delivery to any channel**<br>- **No summary posted to main session**<br>- Job runs silently, output is discarded |

**Key difference:** `announce` sends output somewhere; `none` is completely silent.

---

### 2. Can `delivery.channel` or `delivery.to` be omitted when `mode: "announce"`?

**Yes, they CAN be omitted**, but there's a fallback behavior:

> *"If `delivery.channel` or `delivery.to` is omitted, cron can fall back to the main session's "last route" (the last place the agent replied)."*

However, there's a critical warning:

> *"Missing or invalid delivery targets **fail the job** unless `delivery.bestEffort = true`."*

So:
- **If omitted** ‚Üí Falls back to "last route" (where the agent last replied)
- **If fallback fails/no last route** ‚Üí Job **fails** (unless `bestEffort: true`)
- **Safe approach** ‚Üí Always specify `channel` and `to`, or use `bestEffort: true` as a safety net

---

| Scenario | What Happens |
